# Text_Minning_Code
This Python script is a robust data extraction tool that fetches the latest socioeconomic indicators for all countries from the World Bank API and consolidates them into a clean, structured CSV file. The project demonstrates professional-grade API integration, error handling, and data merging techniques using Python.
The script begins by defining a dictionary of key indicators — population, population growth rate, land area, and population density — mapped to their official World Bank API codes. It then constructs a reusable API template with parameters for JSON output, most recent value (MRV), and a high page limit to retrieve comprehensive data.
The core of the program is the fetch_indicator() function, which is carefully designed for reliability. It sends HTTP GET requests to the World Bank API for each indicator, handles potential failures gracefully, and implements retry logic for rate-limiting errors (HTTP 429). The function parses the complex nested JSON response, extracts relevant fields such as country name, ISO3 code, and indicator value, and returns the data as a clean pandas DataFrame. Empty DataFrames are returned in case of persistent errors, ensuring the script continues running even if one indicator fails.
In the main() function, the script creates a requests Session for efficient connection reuse and iteratively fetches data for all four indicators. Each resulting DataFrame is renamed so that the indicator value column matches its human-readable name. The script then uses functools.reduce() combined with pd.merge() to join all indicator DataFrames on the common keys (iso3 and country) using an outer join. This produces a single comprehensive dataset where each row represents one country with multiple indicators.
After merging, the code performs final data cleaning and formatting: it reorders columns, sorts countries alphabetically, optionally limits the number of rows, and removes any rows containing missing values. The final cleaned dataset is saved as worldbank_country_stats.csv with proper UTF-8 encoding.
The script is production-ready, featuring good practices such as type hints, detailed error handling, retry mechanisms for API rate limits, progress messages, and robust data processing. It effectively transforms raw, complex API responses into a clean, analysis-ready tabular dataset suitable for further use in data analysis, visualization, or machine learning projects.
This tool is particularly valuable because it automates the collection of official World Bank statistics, ensuring the data is always up-to-date with the most recent available values while maintaining high reliability and error resilience.
